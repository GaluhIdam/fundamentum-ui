OidcAuthenticatorService

Description: The OidcAuthenticatorService extends the OidcUtilityService and provides additional functionalities for user authentication and authorization using OIDC.

Key Features:

  • Authentication methods: Provides options for user login with a page or a popup window, offering flexibility and customization in the authentication process.
  • Callback handling: Handles OIDC callback to exchange authorization codes for tokens and stores them securely, ensuring smooth authentication flows and secure token management.
  • Authentication status check: Allows checking the authentication status of the user and retrieving user information, enabling fine-grained control over user authentication and authorization.
  • Logout functionality: Supports logging out by revoking tokens and clearing storage, ensuring secure logout and session management for users.
  • User information retrieval: Retrieves user information using access tokens, allowing seamless integration with OIDC user information endpoints and providing access to user data.

OidcUtilityService

Description: The OidcUtilityService is an Angular service designed to facilitate common tasks related to OpenID Connect (OIDC) authentication and authorization processes.

Key Features:

  • Token management and storage: Provides methods to securely store and retrieve tokens (access, ID, and refresh tokens) using various storage mechanisms such as local storage, session storage, or cookies.
  • Token exchange: Facilitates the exchange of authorization codes for tokens by interacting with OIDC token endpoints, ensuring secure authentication and authorization processes.
  • Cryptographic operations: Generates nonces, code verifiers, and code challenges required for OIDC authentication flows, ensuring cryptographic security in authentication processes.
  • Storage management: Offers methods to set, get, and delete data from local storage, session storage, and cookies, providing flexible and secure storage options for OIDC-related data.
  • Revocation: Supports revoking tokens and clearing storage for logout functionality, ensuring secure logout and session management for users.

Data Transfer Objects:

  • ConfigDTO: Data transfer object for authentication configuration, providing a structured way to pass authentication configuration parameters to OIDC services.
  • WellKnownEndPointDTO: Data transfer object for OIDC well-known endpoints, offering a standardized format for representing OIDC endpoint information and ensuring interoperability with OIDC-compliant systems.

Note: Both services are designed to work seamlessly with Angular applications and abstract complex OIDC-related tasks to simplify OIDC integration and authentication processes.
