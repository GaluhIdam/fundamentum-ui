OidcAuthenticatorService

Overview

The OidcAuthenticatorService is an Angular service designed to streamline authentication and authorization processes using OpenID Connect (OIDC) in Angular applications. It offers a comprehensive set of methods for user authentication, token management, and authentication status checks. This service simplifies OIDC integration by handling token storage, retrieval, and revocation, as well as interaction with OIDC endpoints for authentication and authorization workflows.

Key Features

  • User login and logout functionalities with support for various authentication flows.
  • Token management: storage (local, session, or cookie), retrieval, and revocation.
  • Integration with OIDC endpoints for authentication, token exchange, and user information retrieval.
  • Seamless handling of authentication callback and user session management.

Methods

  1. loginAuth(config: ConfigDTO): Promise<void>
  2. loginWithPopup(config: ConfigDTO): Promise<void>
  3. callBackAuth(config: ConfigDTO): void
  4. checkAuth(config: ConfigDTO): Observable<boolean>
  5. logoutAuth(config: ConfigDTO): Observable<unknown>
  6. getUserInfo(config: ConfigDTO): Observable<unknown>

Configuration

The ConfigDTO interface defines the configuration options required for OIDC authentication, including endpoints, client credentials, and token storage preferences.

OidcUtilityService

Overview

The OidcUtilityService provides utility functions and methods to support OIDC integration in Angular applications. It abstracts common OIDC-related tasks such as token management, cryptographic operations, and HTTP requests to OIDC endpoints. This service facilitates the implementation of OIDC authentication and authorization processes by offering functionalities like code exchange, nonce generation, and token storage management.

Key Features

  • Code exchange for tokens using PKCE (Proof Key for Code Exchange) flow.
  • Generation of nonces, code verifiers, and code challenges for OIDC security.
  • Storage management for tokens (localStorage, sessionStorage, and cookies).
  • HTTP request handling for OIDC endpoints.

Methods

  1. generateNonce(length: number = 16): string
  2. generateCodeVerifier(): string
  3. generateCodeChallenge(verifier: string): Promise<string>
  4. setLocalStorage(key: string, value: string): void
  5. getLocalStorage(key: string): string | null
  6. deleteLocalStorage(key: string): void
  7. setSessionStorage(key: string, value: string): void
  8. getSessionStorage(key: string): string | null
  9. deleteSessionStorage(key: string): void
  10. setCookie(name: string, value: string, days: number): void
  11. getCookie(name: string): string | null
  12. deleteCookie(name: string): void
  13. deleteAllCookies(): void

Configuration

The TokenResponseDTO interface defines the structure of the token response received after code exchange with the OIDC token endpoint.
